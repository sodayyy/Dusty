# Dusty 🧹

> Windows 深度清理助手 · 让普通人也能放心卸干净

面向非技术用户的桌面端 AI Agent，含深度卸载 · 磁盘空间分析 · 智能对话三大模块。  
已发布 Windows 安装包（Tauri v2 + Rust + React）。

---

## 为什么做这个

软件卸载后总觉得"没卸干净"——AppData 里的配置文件还在，注册表碎片还在，有时候控制面板里甚至还能找到它。

这个问题技术用户可以用 Geek Uninstaller 解决，但普通用户不知道该怎么查，也不敢乱删。

Dusty 的定位是：**把"卸干净"这件事做成普通人也敢操作的体验**。三色安全分级、不可逆操作收敛到最后一步、全程有吉祥物解释在做什么。

---

## 主要功能

### 🗑 深度卸载
弹出官方卸载向导（用户自己完成）→ 自动验证是否真的卸载成功 → 扫描三类残留（注册表碎片 / AppData 遗留 / 服务启动项）→ 展示残留清单，用户确认后清理。

### 💿 磁盘空间分析
不只是饼图。同一个页面展示：
- 缓存热点（NVIDIA 着色器缓存 / Windows 临时文件 / 浏览器缓存等 7 条规则，一键清理）
- 大型文件（>100MB 单个文件，按类型标注）
- 可卸载软件（AI 自动打标签：游戏 / 办公 / 开发工具 / 系统组件等 8 类）
- 分类明细饼图（支持点击扇形下钻）

### 🤖 AI 增强层
- **Layer A（静默）**：规则引擎覆盖不到的文件交给 AI 分类，批量合并请求 + 本地缓存，用户无感知
- **Layer B（对话）**：点击吉祥物开启对话，System Prompt 锁定当前上下文，防越界规则 hardcode 在规则层而非依赖 AI 自律

### 🧡 Dusty 吉祥物
4 个状态（idle · sweeping · celebrate · warning）与产品操作阶段联动。语气系统按场景切换：日常活泼 / 危险操作严肃 / 越界固定模板。

## 三次重大转向

产品从初始设想到最终版本经历了三次核心重设计，这是这个项目最值得记录的部分。

### 一、磁盘分析：从「展示数字」到「能做什么」

**初版**：扫描六大类文件，渲染饼图 + 分类数字。

**问题**：「软件程序占 354GB」对用户没有意义——他不知道是哪几个软件，更不知道该删哪个。

**转向**：加入缓存热点（固定规则，不需要扫描）、大型文件列表（BFS 遍历时顺带收集 >100MB 文件）、软件 AI 标签。让用户看到空间真正去哪了——「鸣潮 130GB、赛博朋克 90GB」比「其他类 231GB」有用得多。

---

### 二、卸载流程：发现「静默卸载根本没有运行」

**初版设计**：调用卸载程序时追加 `/S` 静默参数，安静完成，再把残留文件移入暂存区。

**排查过程**：卸载完「小雷游戏盒」后，软件完整存活，注册表没动，安装目录一个文件没少。用 PowerShell 手动运行卸载程序，正常弹出向导。

**根因 A：路径解析错误**

注册表里的 UninstallString 不带引号且路径含空格时：

```
输入：C:\Program Files (x86)\XiaoLeiGamebox\uninst.exe
解析：程序 = "C:\Program"（不存在），参数 = "Files (x86)\..."
结果：Command::new 静默失败，什么都没执行
```

修复：从右往左逐段找第一个真实存在的文件路径，以文件系统为 ground truth，不猜格式。

**根因 B：UAC 权限**

路径修复后仍然没弹向导。原因：装在 Program Files 的卸载程序需要管理员权限，`Command::new` 不触发 UAC，子进程被系统静默拒绝。

**最终结论**：与其追求可靠的静默卸载，不如直接弹出官方向导让用户自己操作。更可靠，用户也更有控制感。UI 改为「官方卸载向导已弹出，请在弹出窗口中完成操作，完成后点击下一步验证」。

---

### 三、暂存区：注册表项无法「移入文件夹」

**初版设计**：卸载后残留文件移入暂存区 `%APPDATA%\Dusty\staging\`，7 天缓冲期，随时可还原。

**发现问题**：即使暂存区运转正常，Geek Uninstaller 里软件仍然「存活」。原因：`HKLM\...\Uninstall\软件名` 这个注册表键只能删或不删，无法被「移动到文件夹」。注册表残留从来没被清理。

**更深的问题**：卸载残留（AppData 配置、注册表碎片）对用户来说本来就没有「还原」的需求——这些是垃圾，不是用户数据。暂存区的设计出发点是「删错了可以恢复」，但对于残留场景，这个需求几乎不存在。

**重设计**：残留展示给用户确认 → 直接删除（文件用 `fs::remove_dir_all`，注册表用 `registry::delete_key`），不走暂存区。用「删之前让用户看清楚」替代「删之后给还原机会」。

## 技术栈

| 层 | 选型 | 说明 |
|----|------|------|
| 桌面框架 | Tauri v2 | 比 Electron 轻 ~10 倍，Rust 原生处理系统操作 |
| 前端 | React + Vite + TypeScript | — |
| 样式 | Inline style + 硬编码色值 | Tailwind v4 在 Tauri WebView 下 class 解析不稳定，全部绕过 |
| 图表 | Recharts | — |
| 状态管理 | Zustand | 扫描缓存提升到 store，切换页面不重新扫描 |
| 后端逻辑 | Rust (Tauri Commands) | 文件 / 注册表 / 进程操作全在 Rust 侧 |
| AI 接口 | DeepSeek API | — |
| 权限 | requireAdministrator manifest | 整体提权，启动时弹一次 UAC，全程有效 |

## 开发方式

这个项目的另一个实验：**非技术背景的产品负责人，通过 AI 工具链独立完成从 0 到 1**。

```
产品决策 / 根因诊断 / 任务拆解
        Claude Desktop（这里）
               ↓
        v0.dev 生成页面原型
               ↓
    Claude Code CLI 接入真实数据和 Rust 逻辑
    （本地运行，DeepSeek API 路由，全权限模式）
               ↓
        tauri build 真实窗口验证
```

三份文档解决 AI 上下文不持久问题：
- `CLAUDE.md`：开发约束与行为规则（CC 每次会话自动读取）
- `DUSTY_SPEC.md`：产品设计规范
- `PHASE5_STATUS.md`：阶段状态，任何时刻重启对话均能无缝恢复

## 安装与运行

**直接使用**

在 [Releases](../../releases) 页面下载 `Dusty_0.1.0_x64-setup.exe` 安装。  
启动时需要管理员权限（UAC 弹窗），这是卸载操作必须的。

**本地开发**

```bash
# 需要：Node.js 18+，Rust 工具链，管理员终端
git clone https://github.com/你的用户名/dusty
cd dusty
npm install
npm run tauri dev   # 必须在管理员终端运行，否则卸载向导不弹
```

> `npm run dev` 下 Tauri invoke() 全部失效属正常，视觉确认在浏览器做，功能联调在 tauri dev 或打包后做。
