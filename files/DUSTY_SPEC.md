## Dusty 吉祥物动画

### 4个状态（SVG帧动画，CSS切换）

| 状态 | 触发时机 | 动作描述 |
|------|----------|----------|
| `idle` | 待机/加载完成 | 小人站立，扫把轻轻晃动 |
| `sweeping` | 扫描中/清理执行中 | 挥动扫把，来回扫地动作 |
| `celebrate` | 清理完成 | 举起扫把，撒星星粒子 |
| `warning` | 重大操作确认/错误 | 举起一只手，表情严肃 |

### 实现方式
- 纯 SVG + CSS `@keyframes`，不引入图片资源
- 每个状态约 3-5 帧，CSS `animation` 切换帧
- 组件：`src/components/Dusty/index.tsx`，接受 `state` prop

## 核心动画清单（Framer Motion）

| 场景 | 动画描述 |
|------|----------|
| 饼图出现 | 扇形从圆心向外展开（`scale` 0→1），数字滚动计数 |
| 点击饼图扇形 | 扇形放大突出 + 右侧滑入二级细分列表 |
| 软件卡片进入清理 | 卡片飞向右下角垃圾桶，垃圾桶盖弹开再合上 |
| 扫描进行中 | Dusty 切换至 `sweeping` 状态 |
| 清理完成 | Dusty 切换至 `celebrate` + 粒子效果 |
| 最终确认弹窗 | 从下方滑入，背景模糊，Dusty 切换至 `warning` |
| 还原操作 | 文件从垃圾桶飞回原位 |
| Dusty 气泡出现 | `scale(0)→scale(1)` + `opacity` 弹出 |
| Onboarding 步骤 | 步骤卡片左右滑动切换 |

## 卸载流程（核心）

```
① 选择目标软件（列表选择 / 拖入exe / 搜索）
        ↓
② 预扫描快照（记录注册表、AppData、服务、开机项）
        ↓
③ 调用官方卸载程序（静默 or 带UI）
        ↓
④ 扫描三类残留（并行）：
   ④-A 注册表残留（HKLM/HKCU 软件键、卸载键）
   ④-B 文件残留（AppData、ProgramData、Temp、用户目录）
   ④-C 系统项残留（服务、计划任务、开机启动项）
        ↓
⑤ 展示残留清单（用户勾选，三色分级）
        ↓
⑥ 用户确认 → 移入暂存区（非真实删除）
        ↓
⑦ 提示"已扫进垃圾桶，7天后自动清除，可随时还原"
        ↓
⑧ 用户主动点"清空垃圾桶" → 最终确认弹窗（严肃语气）
        ↓
⑨ 彻底删除（写rollback日志 → 执行删除 → 更新日志状态）
```

**关键原则：用户在⑦之前任何时候都可以取消，⑧才是不可逆操作。**

## 暂存区设计

- 路径：`%APPDATA%\Dusty\staging\`
- 结构：每次卸载对应一个子文件夹，包含被移入的文件 + `meta.json`
- `meta.json` 记录：原路径、移入时间、软件名、过期时间
- 7天后自动清除（每次启动检查）
- `rollback.json`：记录每步操作，崩溃后重启检测并提示用户

## 文件分类（6大类）

| 类别 | 图标 | 颜色 | 路径特征 |
|------|------|------|----------|
| 软件程序 | 🗂 | 蓝 | Program Files、Program Files (x86) |
| 系统文件 | 🔒 | 灰（禁删） | Windows、System32、SysWOW64 |
| 缓存/临时 | 🗑 | 绿（可删） | Temp、.tmp、AppData\Local\Temp |
| 文档/媒体 | 📄 | 暖黄 | Documents、Desktop、.docx/.jpg等 |
| 安装包 | 📦 | 橙 | .exe/.msi/.iso 在下载目录 |
| 其他 | ❓ | 浅灰 | 无法归类的 |

## 三色安全分级

| 级别 | 颜色 | 含义 | 示例 |
|------|------|------|------|
| ✅ 可直接清理 | 薄荷绿 `#6DBF9E` | 明确的缓存/临时文件 | Temp、浏览器缓存、回收站 |
| ⚠️ 谨慎操作 | 蜂蜜黄 `#F0C070` | 可能有用，不确定 | 大型未访问文件夹、不常用软件数据 |
| 🚫 禁止删除 | 砖红 `#E07060` | 系统文件/关键程序 | System32、驱动、.NET运行时 |

## AI 接口规范

### API Key 管理
- 存储：Tauri 加密 store（`tauri-plugin-store`），不得明文写入任何文件
- 入口：Settings 页，默认隐藏，页面标注"体验版请直接使用，无需填写"
- 开启逻辑：有 Key 且 Key 验证通过 → AI 功能模块显示

### Layer A：分类增强（静默穿插）

触发条件：规则库无法确定分类的文件
调用方式：批量合并，每次最多 30 个文件一起发
结果缓存：同一绝对路径命中缓存，不重复调用
Prompt 模板：
```
你是一个文件分类专家。以下是一批 Windows 文件路径，
请判断每个文件属于哪个类别（软件程序/系统文件/缓存临时/文档媒体/安装包/其他），
以及是否可以安全删除（safe/caution/never）。
只返回 JSON 数组，格式：[{"path":"...","category":"...","safety":"..."}]
文件列表：
{file_list}
```

### Layer B：Dusty 对话（用户主动触发）

System Prompt 模板（每次调用必须注入）：
```
你是 Dusty，一个可爱的 Windows 磁盘清理助手小人。
当前用户正在处理：{software_name}
扫描发现的残留：{residue_summary}
当前操作阶段：{current_stage}

你只能回答与当前清理任务直接相关的问题。
如果用户问其他话题，用以下固定文本回复（不要自由发挥）：
「嘿，Dusty 只懂清理方面的问题哦，其他的我帮不上忙啦～」

遇到重大风险或用户询问是否删除系统文件时，语气必须转为严肃：
「注意：{风险描述}。这类文件删除后可能影响系统稳定，请谨慎决定。」

日常回答保持活泼可爱的语气，句尾可以加「～」或「哦」。
```

### 对话框引导提问（固定展示3条，可点击发送）

根据当前阶段动态切换：

**阶段：已扫描残留**
- 「这些文件都能删吗？」
- 「哪些我必须留着？」
- 「帮我总结一下扫到了什么」

**阶段：磁盘分析**
- 「哪类文件占了最多空间？」
- 「有什么可以马上清理的？」
- 「这个文件夹是干什么用的？」

## Onboarding 引导流程

三步，Dusty 带领：
1. 「嗨！我是 Dusty，你的专属清洁工～先来看看磁盘里有什么吧！」→ 触发磁盘扫描
2. 「找找有没有不需要的软件？」→ 展示软件列表
3. 「准备好了吗？清扫开始！」→ 进入主界面

首次启动显示，之后不再展示（Tauri store 记录）。


存储路径：`%APPDATA%\Dusty\logs\{日期}.md`

## 崩溃保护机制

- 每次删除前写 `rollback.json`，记录操作步骤和状态
- 应用启动时检测 `rollback.json` 是否有 `status: interrupted`
- 若有，Dusty 提示：「上次好像没清理完，要继续还是撤销？」
- 用户选撤销 → 从暂存区还原文件

## 规则库更新机制

- 路径：`rules/` 目录下三个 JSON
- 启动时检查远程版本号（轻量 HTTP GET，失败静默跳过）
- 有新版本 → 后台下载，下次启动生效
- 远程地址：待定（Phase 5 配置）

## UAC 权限提升规范

- 需要提权的操作：注册表 HKLM 写入、系统目录访问、服务删除
- 提权前必须先让 Dusty 说话：「我需要借点权限，马上会弹个确认框，点是就好啦～」
- 使用 Tauri `shell` 插件以 `runas` 方式启动提权子进程
- 提权进程输出通过临时文件传递（见 AGENTS.md 提权规则）
